Pathfinder Prototype

This is a prototype built as part of CSC3068 Software Development Practice, which constitutes the first half of my final year university project. This prototype serves as a proof of concept for a website designed to assist students in selecting their modules and monitoring their degree progress. It achieves this by providing students with a grade dashboard that displays their assignment and module grades, as well as their overall degree grade, making it easy to determine if they are on track for a 2:1 or a 1st class degree. Additionally, users can access a list of all available modules, which they can filter and search through to identify the best modules for their needs. Students who require further guidance can utilize the Pathfinder chatbot, which assists them in the module selection process. Accessibility options have also been implemented to ensure that students with disabilities can easily navigate the website.

This was achieved through the use of Python 3.7.9, Django 3.2, ChatterBot 1.0.8, HTML, SCSS, and JavaScript. The project utilizes an Object-Relational Mapping (ORM) to access the MySQL database, which stores all data, including encrypted user account passwords. Furthermore, a Dockerfile has been included in the project to facilitate ease of deployment.

The files located within the "reports" folder encompass the write-ups, including design documentation, testing reports, and requirements, that were created as part of this module.

A-Level Computer Sciecne Coursework

My Computer Science A-Level coursework was created using Python and SQL and the goal was to create a relational database management system for a dentistry.

I designed the system with a user-friendly graphical user interface (GUI) and ensured that each table had CRUD (Create, Read, Update, Delete) functionality, as well as a robust search feature enabling users to query any field within the tables. Furthermore, I implemented encryption and decryption mechanisms to secure all data within the system. The system also featured a login system with varying access levels for different users.

In addition, I incorporated several advanced features, including the ability to create database backups, generate financial reports that provided graphical insights into the profits of the dental practice over the past year based on the database's information, and the option to save these reports as Word documents. Moreover, the system allowed for seamless email communication with the staff members listed in the database.

All of this was done through the use of various Python libraries.

Technopoly

The Technopoly project was the year long group Coursework project for the Software Engineering and Systems Development module.

The breif for this project was to create a monopoly like game within Java which is to be navigated through menus in a CLI. This involved, creating a design document for the game for the first semester including use case diagrams and descriptions, gantt chart of the development timeline, a class diagram, use case realisations and a boad layout.

In the second semester we then developed the code for the game and created a second report documenting any revisions made, test plan, finalized board layout, and any JUnit test runs which where taken in addition to the testing done in the test plan.

The game itself was (as mentioned before) a monopoly like game where instead of people competing to buy property to gain the most money, it was users competing to use up as much storage space as they can on a hard drive by downloading files. However it isn't just the names that have changed compared to orignal monoply, the Technopoly game also includes mini games like wordle, hangman, tic tac toe and much more.
